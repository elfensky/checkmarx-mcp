# Power BI Setup Guide

Step-by-step instructions for creating a Checkmarx security trends dashboard in Power BI Desktop.

## Prerequisites

- Power BI Desktop (free download from Microsoft)
- CSV data pack generated by `checkmarx.generate-report-data.sh`

## Step 1: Import Data

1. Open Power BI Desktop
2. Click **Get Data** > **Text/CSV**
3. Navigate to `report-data/YYYY-MM-DD/trend-severity.csv` and click **Open**
4. In the preview dialog, verify the columns are detected correctly, then click **Load**
5. Repeat for `trend-new-vs-fixed.csv`

You should now see two tables in the Fields pane: `trend-severity` and `trend-new-vs-fixed`.

## Step 2: Set Column Types

For each table, verify the column types in the Model view:

| Column | Expected Type |
|--------|--------------|
| `granularity` | Text |
| `engine` | Text |
| `period` | Text (not Date — the quarterly format `2026-Q1` won't parse as a date) |
| All numeric columns | Whole Number |

To change a type: select the column in the table view, then use **Column Tools** > **Data Type**.

## Step 3: Add Slicers

Add two slicers to the top of the report page so users can filter the data:

1. Drag `granularity` to the canvas, change visual type to **Slicer**
   - Set to single-select, default to "monthly"
2. Drag `engine` to the canvas, change visual type to **Slicer**
   - Set to multi-select, default to all selected

## Step 4: Create Severity Trend Chart

1. Click an empty area of the canvas
2. Select **Line Chart** from the Visualizations pane
3. Configure:
   - **X-axis:** `period` (from `trend-severity`)
   - **Y-axis:** `high`, `critical` (drag both — they become separate lines)
   - **Legend:** (leave empty — the Y-axis fields act as the legend)
4. Title: "Severity Trend"

### Suggested additional charts

**Stacked bar — all severities:**
- Visual type: **Stacked Bar Chart**
- X-axis: `period`
- Y-axis: `total`
- Legend: add `critical`, `high`, `medium`, `low` as separate values
- This shows the total volume with severity breakdown

**Engine comparison:**
- Visual type: **Clustered Bar Chart**
- X-axis: `engine`
- Y-axis: `total`
- Filter: set granularity slicer to "quarterly" to see per-engine totals

## Step 5: Create Net Change Chart

1. Click an empty area, select **Bar Chart** (clustered)
2. Configure:
   - **X-axis:** `period` (from `trend-new-vs-fixed`)
   - **Y-axis:** `net_change`
3. Title: "Net Change (negative = improvement)"

### Conditional formatting

To color positive bars red and negative bars green:
1. Click the Y-axis field (`net_change`) > **Conditional formatting** > **Background color**
2. Set rules:
   - If value < 0 → green
   - If value >= 0 → red

## Step 6: Create Summary Cards

Add KPI cards to the top of the dashboard:

1. **Current total findings:** Card visual, value = `total` from `trend-severity`, filtered to the most recent period
2. **Net change this period:** Card visual, value = `net_change` from `trend-new-vs-fixed`, filtered to the most recent period
3. **Critical count:** Card visual, value = `critical` from `trend-severity`, filtered to the most recent period

## Step 7: Publish and Share

1. **File** > **Publish** > select your Power BI workspace
2. In Power BI Service (app.powerbi.com):
   - Open the published report
   - Click **Share** to send a link to managers
   - Optionally configure **Scheduled Refresh** if the CSVs are stored in a shared location (OneDrive, SharePoint)

## Refreshing with New Data

When you regenerate the CSV data pack:

1. If using the same output directory: just re-run `generate-report-data.sh`, then **Refresh** in Power BI
2. If using dated directories (`report-data/2026-03-30/`): update the data source path in Power BI:
   - **Transform Data** > **Source** step > update the file path
   - Click **Close & Apply**

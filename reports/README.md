# Checkmarx Trend Reports

This directory contains guides for building Power BI dashboards from the trend data generated by the CLI scripts and MCP tools.

## Quick Start

### 1. Generate the data pack

```bash
# For a single project
./utils/checkmarx.generate-report-data.sh --project-id "your-project-uuid"

# For an entire application
./utils/checkmarx.generate-report-data.sh --application-id "your-app-uuid"

# Tenant-wide
./utils/checkmarx.generate-report-data.sh
```

This creates a `report-data/YYYY-MM-DD/` directory with:
- `trend-severity.csv` — severity counts per engine per period
- `trend-new-vs-fixed.csv` — period-over-period net change per engine
- `metadata.json` — generation parameters and timestamp

### 2. Import into Power BI

See [powerbi-setup.md](powerbi-setup.md) for detailed step-by-step instructions.

### 3. Share

Publish the Power BI report to your workspace and share links with managers.

## CSV Schema

Both CSV files use the same columnar structure so Power BI can slice by granularity and engine using filters/slicers.

### `trend-severity.csv`

| Column | Type | Description |
|--------|------|-------------|
| `granularity` | string | `monthly`, `quarterly`, or `yearly` |
| `engine` | string | `sast`, `sca`, `kics`, `containers`, `apisec`, or `total` |
| `period` | string | `2026-03` (monthly), `2026-Q1` (quarterly), `2026` (yearly) |
| `critical` | integer | Count of CRITICAL severity findings |
| `high` | integer | Count of HIGH severity findings |
| `medium` | integer | Count of MEDIUM severity findings |
| `low` | integer | Count of LOW severity findings |
| `info` | integer | Count of INFO severity findings |
| `total` | integer | Sum of all severities |

### `trend-new-vs-fixed.csv`

| Column | Type | Description |
|--------|------|-------------|
| `granularity` | string | `monthly`, `quarterly`, or `yearly` |
| `engine` | string | `sast`, `sca`, `kics`, `containers`, `apisec`, or `total` |
| `period` | string | Period label (same format as severity) |
| `critical` | integer | Change in CRITICAL count vs previous period |
| `high` | integer | Change in HIGH count vs previous period |
| `medium` | integer | Change in MEDIUM count vs previous period |
| `low` | integer | Change in LOW count vs previous period |
| `info` | integer | Change in INFO count vs previous period |
| `net_change` | integer | Total change across all severities (negative = improvement) |

## Refreshing Data

To update the dashboard with fresh data:

1. Re-run `generate-report-data.sh` (same output directory, or update the path in Power BI)
2. In Power BI Desktop: Home > Refresh
3. If published to Power BI Service: configure a scheduled refresh or refresh manually

## Alternative Consumption

The CSVs work in any tool, not just Power BI:

- **Excel/Google Sheets** — open directly, create pivot tables and charts
- **Grafana** — use the CSV data source plugin
- **Python/pandas** — `pd.read_csv("trend-severity.csv")`
- **Claude (MCP)** — ask Claude to call `generate_report_data` and format the results
